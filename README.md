# EOQC_pq：面向 AI 驱动教学场景的高质量选择题干扰项评估研究

> **Embedding-based Option Quality Control (EOQC)**  
> 面向 AI 驱动教学场景的高质量选择题干扰项评估研究
> Research on the Evaluation of High-Quality Distractors for Multiple-Choice Questions in AI-Driven Teaching

---

## 📌 项目概述

在 AI 驱动的互动教学场景中，自动生成高质量、难度可控的试题是提升教学效果的关键。传统基于大语言模型（LLM）的出题方法在干扰项生成上存在质量不可控、语义分布不合理、与正确答案过度相似或迷惑性不足等问题。

本项目通过语义向量空间分析，构建了一套可量化、可控制的干扰项生成与筛选机制，为自动化出题系统提供可复现的语义评估基准。

---

## 🎯 研究目标

- 评估嵌入向量的余弦相似度在选择题干扰项质量控制与难度分级中的应用价值
- 构建大规模语义关系评估数据集
- 提出基于相似度阈值的干扰项过滤策略与维度选择建议

---

## 📊 数据集

我们构建了一个包含 **9 类核心语义关系**、共计 **1800 对句子** 的评估数据集，每类 200 对样本：

| 关系类型 | 说明 |
|----------|------|
| 同义不同表达 | 语义完全相同，表达方式不同 |
| 双重否定 | 通过双重否定表达肯定含义 |
| 句式变化 | 句子结构改变，核心信息不变 |
| 概念混淆 | 故意替换或混淆相近概念 |
| 近义词 | 使用意思相近但不完全相同的词语 |
| 部分正确 | 包含部分正确信息，整体不准确 |
| 反义词 | 表达与原意相反的含义 |
| 过度概括 | 将特定情况不恰当地推广到一般情况 |
| 无关项 | 内容与题干主题完全无关 |

## 🔬 方法论

### 模型与参数
- **嵌入模型**：Qwen3-Embedding-8B
- **相似度度量**：余弦相似度
- **测试维度**：32、64、128、256、512

### 实验流程
1. 对每对句子计算不同维度下的嵌入向量
2. 计算余弦相似度并汇总统计
3. 分析不同关系类型的相似度分布特征
4. 提出干扰项筛选的阈值和维度建议

---

## 📈 关键发现

### 1. 语义关系相似度排序（由高到低）
1. 双重否定、句式变化、同义不同表达（高相似度，应过滤）
2. 概念混淆、近义词
3. 部分正确、反义词
4. 过度概括
5. 无关项（低相似度）

### 2. 适合作为干扰项的语义关系
- 概念混淆
- 近义词
- 部分正确
- 反义词
- 过度概括

### 3. 应过滤的语义关系
- 双重否定
- 句式变化
- 同义不同表达

### 4. 嵌入维度建议
- **128 维或 256 维** 已能提供足够稳定和精确的语义表征
- 更高维度带来的增益有限，且计算成本增加

### 5. 干扰项筛选阈值建议
- **相似度区间 0.62–0.73** 内的选项最适合作为高质量干扰项
- 可基于该阈值自动过滤低质量候选项

---

## 🛠️ 使用建议

### 推荐流程
1. **指令先行**：通过 Prompt 引导 LLM 生成特定类型（如“概念混淆”）的干扰项
2. **相似度校准**：使用嵌入相似度进行二次筛选
3. **人机协同**：将 AI 作为辅助工具，保持教师的个人风格与教学特点

### 提示工程建议
- 避免使用双重否定、复杂的句式变换和直接的同义改写
- 生成的干扰项必须在主题上紧密相关，但需替换核心概念为同领域接近但不同的词语

---

## ⚠️ 局限性

- 结论基于 Qwen3-Embedding-8B 模型，不同模型可能产生略有差异的相似度绝对值
- 相似度仅为质量评估的一个维度，应结合其他手段（如逻辑合理性、知识覆盖度）协同工作
